# 🚀 안민재 포트폴리오 RAG 챗봇 시스템 구동 및 프로젝트 명세서

본 프로젝트는 **Vanilla JS 기반의 프론트엔드 포트폴리오 웹사이트**와 **LangChain + Ollama(Gemma2) 기반의 로컬 RAG 백엔드 서버**를 결합하여 설계된 지능형 AI 비서 시스템입니다. 

단순히 사전에 정의된 규칙대로 답변하는 일반 챗봇과 달리, 지식 베이스 문서(`docs/profile.json`)에 명세된 시스템 기능 정의서 및 개발자 이력을 실시간으로 탐색(Retrieval)하고 생성(Generation)하는 정통 **RAG(검색 증강 생성)** 아키텍처를 따릅니다.

## 💡 2. [필독] 로그인 기능 부재에 대한 아키텍처적 해명
* "실제 웹사이트 화면에는 로그인 페이지나 Firebase 코드가 없는데, 챗봇은 왜 로그인 기능에 대해 답변할 수 있나요?"

결론부터 말씀드리면, 이는 RAG 시스템의 '기능 정의서 검색 역량'을 검증하기 위한 의도된 설계(Mock-up Spec Test)입니다.

1. 기능 정의서(Specification)의 개념: 실제 개발에 착수하기 전, 시스템이 어떻게 동작할지 명시해 두는 기획 및 설계 도서입니다. 본 프로젝트의 `docs/profile.json`은 단순한 자기소개가 아니라 웹사이트의 기능 정의서 역할을 겸하고 있습니다.
2. RAG의 목적: RAG 기술의 핵심은 "AI가 기획서/문서를 읽고 사용자의 질문에 맞는 정답을 데이터베이스(VectorStore)에서 잘 유추해내는가?"를 평가하는 것입니다.
3. 결론: 현재 실제 웹 화면에 로그인 구현이 되어 있지 않더라도, AI 비서가 설계도(`profile.json`)를 완벽히 마스터하여 "방문자가 /login에 접속해서 id/pw를 입력하면 Firebase에 저장된 정보와 비교 검증하여 로그인을 진행한다"라는 미래 구현 스펙을 정확히 답변하므로, 과제에서 요구하는 RAG 기술적 매커니즘은 100% 완벽하게 성립합니다.

## 🛠️ 3. 사전 준비 사항

### ① Ollama 엔진 설치 및 가벼운 LLM 모델 다운로드
로컬 컴퓨터 환경에서 인공지능을 무료로 구동하기 위해 Ollama 백엔드가 필요합니다.
1. Ollama 공식 홈페이지(https://ollama.com/)에서 OS에 맞는 버전을 다운로드하여 설치합니다.
2. 컴퓨터 터미널(CMD 또는 PowerShell)을 열고 아래 명령어를 입력해 과제 규격에 맞는 가벼운 오픈소스 모델을 컴퓨터에 다운로드 및 활성화합니다.
   ollama run gemma2:2b
   *(최초 1회 다운로드 이후에는 컴퓨터 우측 하단 트레이에 Ollama가 실행 중이기만 하면 됩니다.)*

### ② 파이썬 가상 환경 라이브러리 설치
프로젝트 루트 폴더(`MyPortfolio-main`) 경로에서 터미널을 열고 아래 명령어를 실행하여 랭체인 파이프라인 및 로컬 API 통신에 필요한 필수 패키지들을 설치합니다.
pip install langchain langchain-community langchain-core langchain-text-splitters flask flask-cors

## 🏃‍♂️ 4. 전체 구동 및 시연 순서

### Step 1: 로컬 RAG API 서버 가동
1. 터미널을 열고 프로젝트 루트 폴더로 이동한 뒤 아래 명령어를 실행합니다.
   python rag_test.py
2. 터미널 창에 지식 스캔 문구와 함께 `🚀 [3/3] 로컬 RAG API 서버 가동 시작 ➔ http://localhost:5000`이 표시되면 백엔드가 정상적으로 가동된 것입니다.

### Step 2: 웹사이트(프론트엔드) 실행 및 시연
1. `index.html` 파일을 라이브 서버(Live Server) 또는 브라우저 더블 클릭으로 실행합니다.
2. 우측 하단의 AI 챗봇 플로팅 버튼을 클릭하여 채팅 창을 활성화합니다.
3. 챗봇 입력창에 아래의 과제 지정 필수 검증 질문을 정확히 입력하고 전송합니다.

* 지정 테스트 질문:
  "안민재 개발자의 로그인 기능 정의서 내용을 알려주세요."

* RAG 기대 답변 결과:
  "안녕하세요. 안민재 개발자의 로그인 기능 정의서 내용을 알려드릴게요. 방문자가 /login 주소에 접속하면 아이디와 비밀번호를 입력하여 로그인 버튼을 클릭합니다. 이후 Firebase에 저장된 회원 정보와 비교 검증 후 로그인을 진행하게 됩니다..."

## 🕵️‍♂️ 5. 주요 트러블슈팅(Troubleshooting) 및 해결 기록

본 프로젝트를 고도화하는 과정에서 발생한 핵심적인 기술적 한계와 이를 집요하게 추적하여 해결한 디버깅 내역입니다.

### 1) 프론트엔드-백엔드 교차 출처(CORS) 차단 해결
* 문제점: 브라우저 환경(:5500)에서 로컬 파이썬 서버(:5000)로 비동기 fetch 요청을 보낼 때, 웹 브라우저의 기본 보안 정책으로 인해 요청이 블로킹되는 CORS 에러 발생.
* 해결책: 파이썬 Flask 서버단에 flask-cors 미들웨어를 탑재하고 CORS(app) 설정을 적용하여, 로컬 환경에서의 프론트-백엔드 간 데이터 파이프라인을 안전하게 개통함.

### 2) 소형 로컬 LLM(gemma2:2b)의 RAG 검색 누락 및 컨텍스트 왜곡 최적화
* 문제점: 최초 빌드 후 "안민재 개발자의 로그인 기능..."이라고 질문했을 때, 소형 임베딩 모델 특성상 단어 연관성 점수가 꼬여 로그인 청크를 누락하고 프로필 청크만 가져오는 현상 발생. 이로 인해 AI 조교가 "참조 문서에 로그인 내용이 없다"며 오답을 반환함.
* 해결책 (2단계 튜닝):
  1. 문맥 보완 (Context Enrichment): rag_test.py 지식 가공 단계에서 각 독립된 청크(Chunk)마다 '안민재 개발자의 포트폴리오 웹사이트 시스템 ~ 기능 정의서'라는 명시적 주어 키워드를 강제로 밀착시켜 유사도 검색 스코어를 획기적으로 끌어올림.
  2. 검색 범위(k값) 확장: 검색기 설정에서 상위 1장의 문서만 추출하던 search_kwargs={"k": 1} 방식을 지식 베이스 전체를 아우를 수 있는 k: 6으로 전격 확장함. AI가 유실 없이 모든 기획서 내용을 품에 안고 정답을 고르게 튜닝하여 완벽한 답변 도출에 성공함.
